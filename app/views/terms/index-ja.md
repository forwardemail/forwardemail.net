## サービスレベルアグリーメント（「SLA」） {#service-level-agreement-sla}

### 条項 {#terms-1}

* 「ダウンタイム」とは、サーバー側のエラー率により決定される5％を超えるユーザーエラー率を指します。
* 「月間稼働率」とは、カレンダー月の総分数からその月に発生したダウンタイムの分数を差し引いたものを、カレンダー月の総分数で割ったものを指します。

### 合意 {#agreement}

Forward Emailは、Forward Emailサービスが任意のカレンダー月において有料顧客に対して99.99％の稼働率で利用可能となるよう、合理的な商業的努力（業界標準に準じる）を尽くします。これが達成されない場合、[適格性](#eligibility)に従い、以下に記載の[クレジット](#credits)を受け取る資格がある場合があります。

### クレジット {#credits}

| 月間稼働率               | サービス日数*     |
| ------------------------- | ---------------- |
| < 99.99% – >= 99.90%      | 3                |
| < 99.90% – >= 95.00%      | 7                |
| < 95.50%                  | 15               |

\* サービス日数をアカウントに追加する代わりに、按分返金を選択することも可能です。

### 適格性 {#eligibility}

最低クレジット額は1.00ドル以上である必要があります。最大クレジットは30日分またはダウンタイムに関して直近のカレンダー月にお客様が支払った合計額のいずれか大きい方に制限されます。ダウンタイムの影響を受けた有料顧客は、ダウンタイム発生日から30日以内に[ヘルプリクエスト](/help)を提出してクレジットを申請する必要があります。クレジットはForward Emailの主な管理下にあるダウンタイム要因にのみ適用されます。
